## What is the EM Algorithm?

The **Expectation-Maximization (EM) algorithm** is an iterative optimization method for finding maximum likelihood estimates of parameters in statistical models that involve **latent (hidden) variables**.

It was formalized by Dempster, Laird, and Rubin in their landmark 1977 paper, though the core idea had appeared in earlier work. Today it is used across machine learning, statistics, bioinformatics, speech recognition, and computer vision.

### The Core Problem

Suppose you observe data **X** = {x₁, x₂, …, xₙ}, and you believe it was generated by a mixture of K Gaussian distributions — but you don't know which data point came from which Gaussian. The unknown cluster assignments are the **latent variables** **Z** = {z₁, z₂, …, zₙ}.

You want to find the Gaussian parameters **θ** = {μₖ, Σₖ, πₖ} that maximize the **marginal log-likelihood**:

```
ℒ(θ) = log p(X ; θ) = Σᵢ log Σₖ πₖ · N(xᵢ | μₖ, Σₖ)
```

Directly maximizing this is hard because of the sum inside the log. EM solves this by alternating between two steps.

---

## Key Mathematical Concepts

### Gaussian Mixture Model (GMM)

A GMM assumes the data was drawn from K Gaussian components, each with:

| Parameter | Meaning |
|-----------|---------|
| **μₖ** | Mean vector of cluster k (center in 3D space) |
| **Σₖ** | Covariance matrix of cluster k (shape and orientation of the ellipsoid) |
| **πₖ** | Mixing weight — prior probability of belonging to cluster k |

The probability of a point **x** under the model is:

```
p(x ; θ) = Σₖ πₖ · N(x | μₖ, Σₖ)
```

where N(x | μ, Σ) is the multivariate Gaussian density:

```
N(x | μ, Σ) = (2π)^(-d/2) · |Σ|^(-1/2) · exp(-½ (x−μ)ᵀ Σ⁻¹ (x−μ))
```

### The E-Step — Expectation

Fix the current parameters **θ**. For every data point xᵢ and every cluster k, compute the **responsibility** — the posterior probability that point i was generated by cluster k:

```
r(k | xᵢ) = πₖ · N(xᵢ | μₖ, Σₖ) / Σⱼ πⱼ · N(xᵢ | μⱼ, Σⱼ)
```

This is a direct application of Bayes' theorem. Every point receives a soft assignment across all K clusters, summing to 1. Points close to cluster k get high responsibility for k; distant points get low responsibility.

### The M-Step — Maximization

Fix the responsibilities **r**. Update the parameters to maximize the **expected complete-data log-likelihood** (the ELBO):

```
Nₖ  = Σᵢ r(k | xᵢ)                        (effective cluster size)

μₖ  = (1/Nₖ) Σᵢ r(k | xᵢ) · xᵢ             (weighted mean)

Σₖ  = (1/Nₖ) Σᵢ r(k | xᵢ) · (xᵢ−μₖ)(xᵢ−μₖ)ᵀ   (weighted covariance)

πₖ  = Nₖ / N                                (updated mixing weight)
```

Each quantity is a **weighted average**, where the weight of point i for cluster k is its responsibility r(k | xᵢ).

### Convergence

EM is guaranteed to **never decrease** the log-likelihood. Each E-step and M-step is a coordinate ascent move on a lower bound. The algorithm stops when the change in log-likelihood falls below a threshold ε.

## The ELBO and Why It Matters

The key insight behind EM is the **Evidence Lower BOund (ELBO)**.

Introduce any distribution q(Z) over the latent variables. Then by Jensen's inequality:

```
ℒ(θ) = log p(X ; θ) ≥ ELBO(θ, q)
```

where:

```
ELBO(θ, q) = Σᵢ Σₖ q(zᵢ=k) · [log πₖ + log N(xᵢ | μₖ, Σₖ) − log q(zᵢ=k)]
```

This can also be written as:

```
ℒ(θ) = ELBO(θ, q) + KL(q ∥ p(Z | X, θ))
```

Since KL divergence is always ≥ 0, the ELBO is always ≤ the log-likelihood. The **gap between them is exactly the KL divergence**.

### What Each Step Does to the Bound

| Step | Action | Effect on ELBO | Effect on KL | Effect on LL |
|------|--------|---------------|--------------|--------------|
| **E-Step** | Set q(Z) = p(Z\|X, θ) | ELBO lifts to touch LL | KL → 0 (bound tight) | LL unchanged |
| **M-Step** | Maximize ELBO over θ | ELBO increases | KL re-opens slightly | LL increases |

This interplay is the heart of EM — and it's exactly what the landscape visualization shows.

## What the Visualizations Show

### 1. Cluster Formation View (Left Pane)

This Three.js scene shows the **raw 3D data and the evolving Gaussian fits** in data space.

**Data points** are rendered as glowing spheres. Before initialization they are all grey. As the algorithm runs, each point is colored by its **highest-responsibility cluster** — so points strongly assigned to cluster 1 become blue, cluster 2 becomes pink, and so on.

**Gaussian ellipsoids** represent the 2σ confidence region of each fitted Gaussian. Their shape and orientation come directly from **eigendecomposition of the 3×3 covariance matrix Σₖ** — the three principal axes of the ellipsoid align with the eigenvectors, and their lengths are proportional to √λᵢ (eigenvalues). A spherical Σ produces a sphere; a strongly anisotropic Σ produces a stretched ellipsoid pointing along the direction of highest variance.

**Pulsing mean markers** show the current estimate of μₖ for each cluster.

What to watch:
- After **E-Step**: point colors update — responsibilities have shifted
- After **M-Step**: ellipsoids resize and rotate — the parameters have updated to better fit their assigned points
- Across iterations: ellipsoids shrink and align more tightly with each cluster's actual spread

### 2. LL / ELBO Landscape View (Right Pane)

This scene visualizes the **log-likelihood and ELBO as 3D surfaces over a 2D parameter slice**.

The axes are:
- **X-axis → μ₁** (mean of cluster 1, projected onto the first coordinate)
- **Z-axis → π** (mixing weight of cluster 1)
- **Y-axis → value** (either LL or ELBO)

Every point on the **gold/amber surface** is the log-likelihood ℒ(θ) for that (μ₁, π) combination. The **purple/violet surface** below it is the ELBO for the same parameters — it is a lower bound and therefore always sits at or below the LL surface.

**The amber sphere** marks your current (μ₁, π) position on the LL surface. **The violet sphere** marks the same position on the ELBO surface. The **pink KL pillar** connecting them shows the current KL gap.

**The cyan trail** traces the optimization path through parameter space across all M-steps.

What to watch:
- During **E-Step**: the ELBO surface **animates upward**, lifting until it touches the LL surface at the current parameter point — this is the bound tightening, KL collapsing to zero
- During **M-Step**: the parameter marker slides to a new (μ₁, π), moving toward the peak of the ELBO surface — and since LL ≥ ELBO everywhere, maximizing ELBO also pushes LL upward
- At **convergence**: the trajectory stops moving; the current point is at (or very near) a local maximum of the LL surface

### 3. LL · ELBO · KL Timeline Drawer

The collapsible bottom drawer shows the **history of all quantities across every half-step** (each E-step and M-step is recorded separately, giving twice the resolution of a per-iteration chart).

**Left chart — LL and ELBO over time:**
- Gold line = log-likelihood ℒ(θ)
- Purple line = ELBO(θ, q)
- Pink shaded region = KL gap between them
- Vertical blue dashed lines mark E-steps; pink lines mark M-steps
- The sawtooth-like oscillation of the KL region is the hallmark of EM: it collapses at each E-step and re-opens at each M-step

**Middle chart — KL divergence over time:**
- Shows the pure KL gap as a single curve
- Dramatic drops at every E-step (bound tightens), modest rises after M-steps (bound loosens)

**Right panel:**
- Live numeric values for LL, ELBO, and KL
- **Bound tightness gauge**: ELBO/LL as a percentage — 100% means KL = 0, bound is perfectly tight
- Formula reference for the key identity: ℒ(θ) = ELBO + KL(q ∥ p)

### 4. Elbow Analysis

The sidebar includes an elbow analysis tool that sweeps K from 2 to 8. For each K, it runs EM three times and keeps the best log-likelihood (to reduce sensitivity to local optima), then computes:

| Metric | Formula | Tendency |
|--------|---------|---------|
| **BIC** | p · ln(N) − 2·LL | Penalizes complexity more strongly; prefers simpler models |
| **AIC** | 2p − 2·LL | Lighter penalty; tolerates more clusters |
| **−LL** | −LL | No penalty; always improves with more K |

where p = 10K − 1 is the number of free parameters in a 3D GMM (3 means + 6 unique covariance entries + 1 weight per component, minus the sum-to-1 constraint).

The elbow is detected as the point of **maximum second-derivative curvature** in the score curve — the K where adding one more cluster stops producing proportionally larger gains.

## How to Use the Tool

### Quick Start

1. Click **＋ Sample** to generate 3D data from K synthetic Gaussian clusters
2. Click **⬡ Init** to place K Gaussians at random starting positions
3. Click **▶ E-Step** then **▶ M-Step** alternately to step through the algorithm manually
4. Or click **⟳ Auto** to run continuously (adjust speed with the slider)
5. Click the **◈ LL · ELBO · KL Timeline** tab at the bottom to open the drawer

### Controls Reference

| Control | Effect |
|---------|--------|
| **Sample points** | Number of 3D points to generate |
| **Clusters (K)** | Number of Gaussian components to fit |
| **Convergence ε** | Stop when \|ΔLL\| < ε. Lower = more precise, slower |
| **Ellipsoid σ** | How many standard deviations the ellipsoid surfaces represent |
| **Ellipsoids / Wire / Pts** | Toggle between solid, wireframe, or point-only display |
| **Speed** | Auto-run delay from 700 ms (1×) down to 40 ms (5×) |
| **◈ Run Elbow** | Sweep K=2…8, show BIC/AIC/LL curve with elbow marker |

### Navigation (Both 3D Views)

| Gesture | Action |
|---------|--------|
| Left-drag | Orbit camera |
| Right-drag | Pan camera |
| Scroll | Zoom in/out |

## Algorithm Reference

### Initialization

Cluster means are seeded at K randomly selected data points. Covariances are initialized to scaled identity matrices. Mixing weights are uniform (1/K).

### E-Step (Expectation)

For each point xᵢ and cluster k:

```
rᵢₖ = πₖ · N(xᵢ | μₖ, Σₖ) / Σⱼ πⱼ · N(xᵢ | μⱼ, Σⱼ)
```

### M-Step (Maximization)

```
Nₖ = Σᵢ rᵢₖ
μₖ = Σᵢ rᵢₖ xᵢ / Nₖ
Σₖ = Σᵢ rᵢₖ (xᵢ − μₖ)(xᵢ − μₖ)ᵀ / Nₖ     (diagonal clamped at 200 for stability)
πₖ = Nₖ / N
```

### ELBO

```
ELBO = Σᵢ Σₖ rᵢₖ · [log πₖ + log N(xᵢ | μₖ, Σₖ) − log rᵢₖ]
```

### Log-Likelihood

```
ℒ(θ) = Σᵢ log Σₖ πₖ · N(xᵢ | μₖ, Σₖ)
```

### Convergence Criterion

```
|ℒ(θ_new) − ℒ(θ_old)| < ε   (after iteration 3+)
```

## Technical Implementation

| Aspect | Detail |
|--------|--------|
| **3D rendering** | [Three.js r128](https://threejs.org/) — two independent WebGL renderers sharing a single DOM page |
| **Cluster scene** | Custom orbit controls, Phong shading, ellipsoids via eigendecomposition + Matrix4 transform |
| **Landscape scene** | 44×44 grid surfaces with vertex color maps (amber = LL, violet = ELBO), animated ELBO mesh during E-step |
| **Eigendecomposition** | Jacobi iteration (100 sweeps) for symmetric 3×3 matrices |
| **3D Gaussian PDF** | Full Mahalanobis distance: exp(−½ dᵀ Σ⁻¹ d) / ((2π)^(3/2) √|Σ|) |
| **Landscape axes** | X = μ₁ (first cluster mean, x-component), Z = π₁ (mixing weight), Y = LL or ELBO value |
| **ELBO animation** | Per-frame linear interpolation between old and new vertex Y-positions with ease-in-out |
| **Elbow detection** | Maximum second-derivative curvature: argmax_k (score[k−1] + score[k+1] − 2·score[k]) |
| **Typography** | Playfair Display (headers), DM Mono (code/values), DM Sans (body) |
